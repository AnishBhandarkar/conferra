# 🎯 Project Overview

A **Tech Event Management Platform** where:

* Users can browse tech events
* View event details
* Register & login
* Book events
* View their bookings
* Admins can create / edit / delete events

Think of it like a mini Eventbrite, but focused on tech meetups.

---

# 👤 Roles

## 1️⃣ User

* Register
* Login
* Browse events
* Book event
* Cancel booking
* View “My Bookings”

## 2️⃣ Admin

* Everything user can do
* Create event
* Edit event
* Delete event
* View all bookings
* Manage event capacity

Keep roles simple: `"USER"` and `"ADMIN"`

---

# 🔐 Authentication Requirements

Yes, auth is required.

### Auth System Should:

* Use JWT (stored in HTTP-only cookies)
* Support Register & Login
* Password hashing (bcrypt)
* Middleware route protection
* Role-based access control
* Logout (clear cookie)

### Protected Routes:

* Create Event → Admin only
* Book Event → Logged-in user
* My Bookings → Logged-in user
* Admin dashboard → Admin only

---

# 📅 Event Model Requirements

Each Event should have:

* title
* description
* date
* time
* location
* category (AI, Web, Cloud, DevOps, etc.)
* speaker name
* max capacity
* current booked count
* price (optional)
* createdBy (admin id)
* timestamps

Add proper indexing on:

* date
* category

---

# 🎟 Booking Model Requirements

Each Booking should:

* reference user
* reference event
* booking status (CONFIRMED / CANCELLED)
* timestamps

Constraints:

* A user cannot book same event twice
* Booking blocked if event is full
